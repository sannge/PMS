# Tips & Keyboard Shortcuts

Maximize your productivity with PM Desktop using these power-user features, keyboard shortcuts, and tips.

---

## AI Assistant (Blair)

Blair is PM Desktop's built-in AI assistant. You can use natural language to search your data, manage your work, and get answers.

### What Blair Can Do

Blair has access to a wide range of tools:

| Category | Examples |
|----------|----------|
| **Search** | Search your knowledge base, find documents, look up tasks |
| **Read** | Summarize documents, review task details, check project status |
| **Create** | Create tasks, add comments, create checklists |
| **Update** | Update task status/priority/assignees, edit documents |
| **Delete** | Delete tasks, documents (with your confirmation) |
| **Export** | Export documents to PDF |
| **Web** | Search the web, fetch webpage content |
| **Manage** | Create/update applications, projects, and members |

### Human-in-the-Loop Confirmation

For all write operations (creating, updating, or deleting anything), Blair asks for your confirmation before making changes. You review the proposed action and approve or reject it. This means Blair never makes changes without your explicit approval.

### Example Conversations

- "What documents do we have about the authentication system?"
- "Create a high-priority bug task in project AUTH called 'Fix session timeout'"
- "What tasks are assigned to me in the Product Suite application?"
- "Add a comment to SP1-15 saying 'Ready for review'"
- "Summarize the deployment guide document"
- "Export the API specification document as PDF"

### Chat Sessions

- Chat sessions persist across application restarts
- You can start new conversations or continue previous ones
- Use the back button to return to the session list
- Time-travel: you can go back to any previous point in a conversation and branch from there

### Tips for Using Blair

- Be specific about what you want -- include project names, task keys, or document titles when possible
- Blair respects your role-based permissions, so it can only do what your role allows
- For complex requests, break them into steps
- Blair can search using semantic meaning, not just keywords -- ask questions in natural language

## Productivity Tips

### Quick Actions

1. **Dashboard shortcuts**: Use Quick Action buttons for fast navigation
2. **Breadcrumb navigation**: Click any level to jump back
3. **Double-click**: Edit items inline where supported

### Efficient Task Management

1. **Keyboard navigation**: Arrow through tasks without mouse
2. **Quick create**: Type and press Enter immediately
3. **Use Blair**: Ask the AI assistant to create or update tasks

### Comment Efficiency

1. **@Mention dropdown**: Start typing name after @ for quick selection
2. **Paste images**: Directly paste screenshots from clipboard
3. **Markdown shortcuts**: Use formatting marks instead of toolbar
4. **Threaded replies**: Reply to specific comments to keep discussions organized

### Note-Taking Speed

1. **Tab to save time**: Open multiple documents, switch with tabs
2. **Keyboard formatting**: Faster than clicking toolbar buttons
3. **Application switcher**: `Cmd/Ctrl + K` for instant switching
4. **Tags**: Use tags to categorize and quickly filter documents

## Search Tips

### Effective Searching

1. **Be specific**: Use unique terms for better results
2. **Use filters**: Combine search with filters
3. **Search by key**: Task keys (e.g., "SP1-42") for direct access
4. **Use Blair**: Ask the AI assistant for semantic search across documents

### Search Patterns

| Pattern | Finds |
|---------|-------|
| `bug` | All items containing "bug" |
| `login fix` | Items with both "login" and "fix" |
| `SP1-*` | All tasks in project SP1 |

### Knowledge Base Search

The knowledge base search supports:
- **Keyword search**: Finds exact and partial matches
- **Typo-tolerant search**: Finds results even if you misspell words
- **Semantic search**: Finds conceptually related content (e.g., searching "authentication" finds documents about "login" and "credentials")

## Canvas and Diagram Tips

### Working with Draw.io Diagrams

1. **Use templates**: Draw.io includes built-in templates for common diagram types
2. **Keyboard shortcuts within Draw.io**: Standard Draw.io shortcuts work within the embedded editor
3. **Export options**: Diagrams can be exported as images from within the editor
4. **Preview images**: Diagrams generate PNG previews automatically for quick viewing in documents

### When to Use Diagrams

- Architecture and system design documentation
- Process flows and decision trees
- Wireframes and UI mockups
- Organizational charts
- Network topology
- Database schema diagrams

## Document Import Tips

### Preparing Files for Import

1. **Clean up content**: Remove unnecessary formatting before importing
2. **Use supported formats**: PDF, DOCX, PPTX, XLSX, VSDX
3. **Organize first**: Place files in a logical folder structure before batch importing
4. **Check file sizes**: Keep individual files under 100 MB

### After Importing

1. **Review extracted content**: Imported content may need minor formatting adjustments
2. **Add tags**: Tag imported documents for easier filtering
3. **Verify search**: Confirm documents appear in search results (check the sync badge)
4. **Edit as needed**: Use the rich text editor to refine imported content

### Modals and Panels

1. **Click outside**: Often closes the modal

3. **Enter to submit**: Quick confirmation

### Theme
