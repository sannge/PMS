## Quick Overview

### Application Hierarchy

PM Desktop organizes work in a three-level hierarchy:

```
Applications (Top Level)
    └── Projects (Mid Level)
            └── Tasks (Work Items)
```

- **Applications**: Top-level containers representing major initiatives, products, or departments
- **Projects**: Groupings of related work within an application (e.g., sprints, features)
- **Tasks**: Individual work items with status, assignees, checklists, and comments

### Key Features

| Feature | Description |
|---------|-------------|
| Kanban Board | Visual task management with drag-and-drop status updates |
| Real-time Collaboration | See team members' presence and get instant updates |
| Rich Comments | Formatted text with @mentions and file attachments |
| Checklists | Track sub-tasks and requirements within tasks |
| Knowledge Base | Rich-text notes with document locking, full-text search, and folder organization |
| Role-based Access | Owner, Editor, and Viewer permission levels |

### Navigation

The main sidebar provides quick access to all sections:

- **Dashboard**: Overview with statistics and quick actions
- **Applications**: Browse and manage all applications
- **Projects**: Direct access to project list
- **Notes**: Knowledge base with hierarchical documents
- **Settings**: Application preferences (coming soon)

## System Requirements

- **Operating System**: Windows 10/11, macOS 10.15+, or Linux (Ubuntu 20.04+)
- **Memory**: 4GB RAM minimum, 8GB recommended
- **Display**: 1280x720 minimum resolution
- **Network**: Stable internet connection for real-time features
